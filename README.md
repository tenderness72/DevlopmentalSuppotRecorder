# Cairn（ケルン）

発達支援セッションの記録を一つひとつ積み重ね、その子の歩みを道標として残すための Windows デスクトップアプリ。

> **名前の由来**：Cairn（ケルン）はスコットランド/ケルト由来の「登山道の石積み道標」。日々のセッション記録を積み重ねることで、その子の発達の道を後から振り返り、次の一歩を示せる、という願いを込めています。

---

## 想定ユーザー

- 児童発達支援・放課後等デイサービスのスタッフ
- ABA/療育セッションを個別に記録・分析したい支援者
- 個人利用（1 PC = 1 ユーザー）を想定。データは PC ローカルに保存されます

---

## 主な機能

### 児童管理
- 児童マスタの登録・編集・削除（ソフトデリート）
- 児童コード（K001, K002…）の自動採番
- 大田ステージ、診断、開始日、目標スキル等の属性管理
- **CSV インポート**（後述）
- 児童別 Excel 出力（セッション + 自然場面記録のサマリ）

### プログラム管理
- 介入プログラムのマスタ管理
- **領域（マンド、ToM、感情弁別 等）** ／ **型（構造化WS、ロールプレイ 等）** を別マスタとして編集可能
- 達成基準・備考の管理
- 領域マスタ・型マスタもアプリ画面から追加・編集・削除可能

### セッション記録
- 児童 × プログラム × 日付ごとのセッション記録
- 試行数・正答数・正答率の自動計算
- 習得レベル、プロンプトレベル（独立/視覚/言語/身振り/モデリング/身体）
- 臨床メモ、仮説、次回検証課題
- 試行数・正答数の整合性チェック（正答 > 試行を防止）

### 自然場面記録
- 日常場面での観察記録
- 状況・観察された行動・結果・臨床的解釈・次回検証課題

### 一覧・分析
- セッション一覧（児童・期間フィルタ、CSV/Excel 出力）
- 自然場面一覧（同様）
- **進捗グラフ**：児童＋プログラム単位で正答率の推移を可視化、PNG 画像として保存可能
- 横断検索：臨床メモ・観察行動等の全文検索

### 安全機能（フールプルーフ）
- 未保存変更がある状態での画面遷移・終了時に確認ダイアログ
- 削除操作はすべて「キャンセル」がデフォルト
- 起動時に毎日 1 回 DB を自動バックアップ
- 手動バックアップ・任意の場所への DB エクスポートも可能

### データ保存場所
- DB：`%LocalAppData%\SessionRecorder\session_records.db`
- 自動バックアップ：`%LocalAppData%\SessionRecorder\backups\`
- アプリをアンインストールしても DB は残ります（再インストールで継続利用可能）

`installer\Release\installer.msi` をダブルクリックして実行。

## 児童マスタ CSV インポート

「児童一覧」画面の **「📥 CSVインポート」** ボタンから取り込めます。

### ファイル形式
- 文字コード：**UTF-8**（BOM ありでも無しでも OK）
- 区切り：**カンマ**（`,`）
- 改行：CRLF / LF どちらでも OK
- 値にカンマや改行を含む場合は **ダブルクォート `"…"` で囲む**（CSV 標準）
- 値内に `"` を含めたい場合は `""` とエスケープ

### ヘッダ行（任意）
1 行目が以下のいずれかなら、ヘッダとしてスキップされます：
- `ChildCode,...`
- `児童コード,...`

ヘッダが無くてもインポート可能です（その場合 1 行目からデータとして扱われます）。

### 列の構造

| # | 列名 | 必須 | 型 / 例 | 説明 |
|---|---|---|---|---|
| 1 | ChildCode（児童コード） | ✅ | `K001` | 一意のコード。既存と重複する行はスキップされる |
| 2 | Name（氏名） | ✅ | `山田太郎` | 児童の表示名 |
| 3 | Gender（性別） | ⭕ | `男` / `女` / `その他` / 空 | 自由テキスト |
| 4 | BirthDate（生年月日） | ⭕ | `2018-04-01` / `2018/04/01` / 空 | 文字列として保存（書式は自由） |
| 5 | PrimaryDiagnosis（主診断） | ⭕ | `ASD` / `ADHD` / 空 | 自由テキスト |
| 6 | OtaStage（大田ステージ） | ⭕ | 下表参照 / 空 | 認められないステージ表記は空扱い |
| 7 | StartDate（支援開始日） | ⭕ | `2024-04-01` | `DateTime.TryParse` が成功する形式 |
| 8 | CurrentTargetSkills（現在の目標スキル） | ⭕ | `要求行動の獲得` | 自由テキスト |
| 9 | Notes（備考） | ⭕ | `家庭との連携が必要` | 自由テキスト |

✅ = 必須、⭕ = 任意

#### OtaStage（大田ステージ）の受付表記

| CSV に書く値 | 解釈 |
|---|---|
| `Ⅰ` または `1` | Stage1 |
| `Ⅱ` または `2` | Stage2 |
| `Ⅲ-1` または `3-1` | Stage3-1 |
| `Ⅲ-2` または `3-2` | Stage3-2 |
| `Ⅳ-前` または `4-前` | Stage4-前 |
| `Ⅳ-後` または `4-後` | Stage4-後 |
| `未測定` または 空 | 未測定 |

### サンプル CSV

```csv
ChildCode,Name,Gender,BirthDate,PrimaryDiagnosis,OtaStage,StartDate,CurrentTargetSkills,Notes
K001,山田太郎,男,2018/04/01,ASD,Ⅲ-1,2024/04/15,要求行動の獲得,家庭との連携を重視
K002,佐藤花子,女,2019/06/10,ADHD,Ⅳ-前,2024/05/01,"指示の理解、待つ行動",
K003,鈴木次郎,男,2017/12/20,,2,2023/10/01,共同注意の確立,"カンマ,を含む備考もOK"
```

### インポート挙動

- **重複コードはスキップ**：同じ `ChildCode` を持つ児童が既に登録されている行は読み飛ばされます（既存データを上書きしません）
- **必須列が空の行はエラー**として警告に表示され、その行はスキップ
- **列数が 2 未満**の行はエラー
- 完了後、`インポート完了: N件追加、M件スキップ` のメッセージが表示され、エラーがあれば最大 10 件まで一覧表示

### 注意点

- 列の順序は厳密です（ChildCode → Name → Gender → … の順）
- 空欄を表すには **連続カンマ**（例：`K001,山田太郎,,,,,,,,`）
- Excel から「CSV UTF-8 (.csv)」で保存すれば文字化けせず取り込めます

## バックアップ・データ復旧

- 起動時自動バックアップ：1 日 1 回、`backups\` フォルダに `session_records_YYYYMMDD.db` 形式で保存
- 手動バックアップ：「バックアップ」画面から作成
- DB エクスポート：別 PC への移行や保管用に任意の場所へコピー可能
- 復旧：エクスプローラーで `%LocalAppData%\SessionRecorder\` を開き、`session_records.db` を バックアップで上書きするだけ

---

## 配布・インストーラ作成（開発者向け）

`installer\installer.vdproj` を Visual Studio で開いてリビルドすると `installer\Release\installer.msi` が生成されます。
ProductVersion をバンプ（例：`1.1.0` → `1.1.1`）してから再ビルドすると、既存ユーザーの上書きインストールが正常に動作します。
